# OpenClaw Self-Trainer

让 OpenClaw 的 AI 助手从自己的对话中学习，逐步用本地小模型替代昂贵的云端大模型调用。

## 核心理念

OpenClaw 平时正常工作，后台默默收集对话数据、定期训练、自动评估、渐进式地把越来越多的请求卸载给本地小模型。

```
用户请求 → [路由器] → 小模型(本地) / 大模型(云端) → 回复用户
                    │
                    └── 持续学习：收集日志 → 训练 → 评估 → 更新路由规则
```

## 架构

### 四个核心模块

1. **数据管道** — 从 OpenClaw session 日志收集、清洗、分类对话数据
2. **SFT 训练** — 用清洗后的数据微调本地小模型
3. **自动评估** — 对比新旧模型，客观评估训练效果
4. **智能路由** — 根据评估结果决定哪些请求可以分发给小模型

### 闭环流程

```
┌─────────────────────────────────────────────┐
│                 OpenClaw (运行时)             │
│   用户 → [路由器] → 小模型/大模型 → 回复用户   │
│   所有交互 → 写入日志                         │
└──────────────────┬──────────────────────────┘
                   │ 日志
                   ▼
┌─────────────────────────────────────────────┐
│              数据管道 (定时跑)                │
│   原始日志 → 清洗 → 去重 → 分类 → 训练集     │
└──────────────────┬──────────────────────────┘
                   │ 干净数据
                   ▼
┌─────────────────────────────────────────────┐
│              训练 (有新数据时触发)             │
│   训练集 → SFT 微调 → 新模型 checkpoint     │
└──────────────────┬──────────────────────────┘
                   │ 新模型
                   ▼
┌─────────────────────────────────────────────┐
│              评估 + 决策 (训练后跑)            │
│   新模型 vs 旧模型 vs 大模型                  │
│   → 通过 → 部署                              │
│   → 不通过 → 继续收集数据                     │
│   → 退步了 → 回滚到上一个版本                  │
└─────────────────────────────────────────────┘
```

## 评估体系

不依赖人工评价，基于客观指标：

- **任务成功率** — 是否调用了正确的工具？结果是否正确？
- **幻觉检测** — 是否编造了不存在的信息？
- **工具选择准确率** — 是否选对了工具
- **指令遵循度** — 是否正确理解并执行了用户指令

## 路由器

基于评估结果动态更新路由规则：

```
第一轮训练后：
  工具操作: 85% 准确  → 路由给小模型
  生活闲聊: 91% 准确  → 路由给小模型
  复杂推理: 38% 准确  → 大模型兜底

更多数据训练后：
  复杂推理: 55% 准确  → 继续收集，还不行

最终目标：
  越来越多的请求被小模型处理
  大模型只处理最难的部分
  API 成本线性下降
```

## 时间线

| 阶段 | 时间 | 内容 |
|------|------|------|
| 搭管道 | 第1周 | 清洗脚本、数据集管理、评估脚本 |
| 纯收集 | 第2周 | 只收集数据，不训练 |
| 首次训练 | 第3周 | 第一次 SFT + 评估 |
| 渐进路由 | 第4周 | 部署，只路由最简单的请求 |
| 闭环运行 | 第5-8周 | 逐步扩大路由范围 |
| 稳态运行 | 第2-3月 | 大部分简单请求由小模型处理 |

## 预期效果

- API 成本降低 40-60%
- 简单任务响应更快（本地推理）
- 用户只需偶尔看看周报

## 快速开始

```bash
# 1. 克隆 repo
git clone https://github.com/PawnOfDelock/openclaw-self-trainer.git

# 2. 让 OpenClaw 读 SETUP.md 并自动配置
# 对 OpenClaw 说："帮我安装 openclaw-self-trainer"
```

OpenClaw 会自动读取 SETUP.md，引导完成环境检查、依赖安装、cron 配置等全部流程。
